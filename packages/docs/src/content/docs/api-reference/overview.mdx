## Primary entry points

The package exposes two primary usage styles:

1. `prompt()` for one-shot requests
2. `createSession()` / `resumeSession()` / `forkSession()` for stateful workflows

```ts
import { prompt, createSession, resumeSession, forkSession } from 'open-agent-sdk';
```

## One-shot API

`prompt(input, options)` returns:

- `result: string`
- `duration_ms: number`
- `usage: { input_tokens; output_tokens }`
- `session_id?: string`
- `structured_output?: unknown`

## Session API

Session lifecycle:

- `await session.send(message)`
- `for await (const event of session.stream()) { ... }`
- `session.getMessages()` for history
- `await session.close()`

## Other important exports

- Providers: `OpenAIProvider`, `GoogleProvider`, `AnthropicProvider`
- Permissions: `PermissionManager`, `PermissionMode`, `CanUseTool`
- Tools: built-in tool classes and input/output types
- Hooks: `HookManager` and typed hook payloads
- Output format: `OutputFormat`, `Schema`
