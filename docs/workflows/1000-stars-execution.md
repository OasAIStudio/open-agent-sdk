# 1000+ Stars Execution Plan (60-90 Days)

This document translates growth strategy into weekly execution inside this repository.

## North Star and KPIs

- North Star: weekly net stars
- Activation: quickstart issue rate (quickstart-related issues per week)
- Retention proxy: external repository mentions / inbound links

## Operating Rhythm

Weekly cadence:
- Tuesday: ship one demo artifact (code + screenshot/video + command)
- Thursday: publish one short engineering write-up with repo link
- Weekend: submit to two distribution channels (HN/Reddit/awesome lists/communities)

## Week 1 (Positioning + README Funnel)

Goals:
- Make first 30 seconds clear
- Make first 1 minute runnable

Ship checklist:
- [x] Pin one-line positioning in root README
- [x] Add 1-minute quickstart in root README
- [x] Link 30-second visual demo
- [x] Add demo gallery page
- [x] Add benchmark entry page

Definition of done:
- New visitor can identify value proposition and first command from root README without leaving the page.

## Week 2-3 (Eval as Citation Asset)

Goals:
- Turn eval into reusable citation material

Ship checklist:
- [ ] Run SWE-bench Lite batch with fixed config
- [ ] Publish raw artifacts under `benchmark/swebench/outputs/`
- [ ] Add dated results section in `BENCHMARKS.md`
- [ ] Add reproducibility notes (env/model/turn limits)

Definition of done:
- Another engineer can rerun benchmark command and produce compatible report files.

## Week 4-6 (Distribution + Integrations)

Goals:
- Build external traffic sources

Ship checklist:
- [ ] Prepare 10 integration PR targets
- [ ] Contribute adapter/examples PRs
- [ ] Track merged PR backlinks to this repo
- [ ] Submit to 2+ relevant awesome lists

Definition of done:
- At least 3 external references merged and pointing back to this repository.

## Demo Quality Bar

Each public demo should include:
- Single command to run
- Clear input and output path
- 20-30 second recording
- Copyable snippet for social post
